# Client Review

Prepare a meeting-ready client review that is clear, calm, and practical. Focus on what the client cares about, what changed since the last meeting, what requires action now, and what should wait.

## Workflow

### 1. Rebuild the client context

Start with the household, not the portfolio.

Gather or confirm:
- Client / household names
- Life stage: accumulation, pre-retirement, retirement, legacy, liquidity event, etc.
- Account types and approximate total assets
- Target allocation or IPS constraints
- Last meeting date
- Outstanding action items
- Relevant personal updates: job, income, spending, family, health, tax, estate

If prior notes are incomplete, list the missing context explicitly.

### 2. Summarize performance at the right level

Prepare household-level performance first, then drill into accounts only where it helps explain the story.

Useful views:
- QTD / YTD / 1Y performance
- Benchmark comparison
- Top contributors and detractors
- Any unusually concentrated position impact
- Any recent cash flows, withdrawals, or contributions affecting interpretation

Do not bury a weak period. Explain it simply.

### 3. Review allocation drift

Compare current allocation against target ranges.

Use a compact table:

| Asset class | Target | Current | Drift | Action |
|---|---|---|---|---|
| Equities | | | | |
| Fixed income | | | | |
| International | | | | |
| Alternatives | | | | |
| Cash | | | | |

Flag drift that is large enough to matter, not every tiny variance.

### 4. Prepare the meeting agenda

Default agenda:
1. What changed since last meeting
2. Portfolio performance and why
3. Allocation review and rebalancing needs
4. Planning updates: spending, retirement, tax, estate, liquidity
5. Decisions and next steps

Keep the sequence human: life first when appropriate, markets second.

### 5. Generate recommendations

Recommend only actions that are timely and explainable.

Typical actions:
- rebalance positions
- harvest losses or gains intentionally
- deploy idle cash
- adjust withdrawal plan
- review beneficiaries or estate documents
- revisit insurance coverage
- schedule deeper planning work on retirement, college, or concentrated stock

For each action, include the reason and urgency.

### 6. Deliver the review pack

If the user wants a consistent meeting memo or action-item sheet, read `references/output-template.md` before drafting.

Default output:
- one-page client review summary
- 5 or fewer talking points
- 3-5 action items with owners and next dates

If the user wants more, expand into a fuller memo or presentation.

## Output standard

Default to a concise meeting-prep memo, not a long handbook.
Unless the user explicitly asks for more depth, cap the default output at:
- 1 short opening summary paragraph
- 5 or fewer talking points
- 1 allocation review table
- 3-5 recommended actions
- 3 or fewer open questions for the client

Do not include extended scripts, multiple alternate scenarios, or long optional frameworks by default.
Read `references/output-template.md` when the user wants a meeting memo, action sheet, or review summary, and follow that structure tightly.

The client review should help the advisor enter the meeting knowing:
- what matters most to this client right now
- what changed since the last review
- what to say about performance
- whether any portfolio action is needed
- what decisions to ask for before the meeting ends

## Quality bar

- Lead with the client, not the market.
- Keep jargon low unless the client is highly sophisticated.
- Make recommendations proportional to the issue.
- Be direct if performance disappointed.
- End with specific next steps, dates, and owners.
- Separate factual recap from advice or recommendations.
